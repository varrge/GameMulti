# 开发排期与任务拆分（粗版）

## 1. 里程碑总览

建议按以下 6 个里程碑推进：

1. 基础规划与脚手架
2. 主站身份与邀请制模型 MVP
3. 游戏绑定闭环与论坛打通
4. 金币结算与商城兑换
5. 后台管理与封禁联动
6. 多游戏扩展与上线准备

---

## 2. Milestone 1：基础规划与脚手架（第 1 周）

### 目标
建立技术基线、目录结构、基础文档与开发协作约定。

### 交付内容
- 技术选型定稿
- 仓库结构确认
- 分支管理规范
- 前端脚手架初始化
- 后端脚手架初始化
- 数据库建模初稿
- 插件通信协议初稿
- Discourse 选型确认

### 依赖
无

---

## 3. Milestone 2：主站身份与邀请制 MVP（第 2~3 周）

### 目标
跑通主站用户、邀请码、注册登录与用户中心基础框架。

### 前端任务
- 官网首页
- 登录/注册页面
- 邀请码输入流程
- 用户中心基础页

### 后端任务
- `users` / `invitation_codes` / `invitation_code_usages` 模型实现
- 登录注册 API
- 邀请码校验逻辑
- 用户中心 API

### 验收标准
- 用户可通过邀请码完成注册
- 用户可登录主站
- 后台可追踪用户邀请来源

---

## 4. Milestone 3：绑定闭环与论坛打通（第 4~5 周）

### 目标
跑通“游戏内发起绑定 -> Web 确认 -> 主站正式绑定 -> 论坛身份同步”闭环。

### 前端任务
- 绑定确认页
- 配对码输入页
- 绑定结果页
- 论坛导航融合

### 后端任务
- `binding_sessions`
- `game_accounts`
- `user_game_bindings`
- 绑定确认 API
- 主站 SSO 中心
- Discourse 打通

### 插件任务
- 首个 PoC 游戏插件支持 `/register` / `/bind`
- 绑定链接与配对码创建

### 验收标准
- 玩家可在游戏内发起绑定
- 玩家可在网页完成确认
- 绑定结果可在用户中心查看
- 主站登录后可进入论坛

---

## 5. Milestone 4：金币结算与商城兑换（第 6~8 周）

### 目标
跑通玩家行为上报、奖励结算、钱包入账、商城兑换和发货任务。

### 前端任务
- 服务器状态展示
- 钱包页与流水页
- 商城列表
- 兑换订单页

### 后端任务
- `game_player_events`
- `game_play_sessions`
- `reward_rules`
- `coin_reward_settlements`
- `wallets` / `wallet_transactions`
- `redeem_orders` / `reward_delivery_jobs`

### 插件任务
- 上报玩家行为事件
- 上报服务器状态
- 拉取发奖命令

### 验收标准
- 指定行为可稳定转为金币
- 金币流水可追踪
- 用户可成功兑换至少一种商品
- 异步发货链路可重试

---

## 6. Milestone 5：后台管理与封禁联动（第 9~10 周）

### 目标
补齐运营后台与封禁联动能力。

### 前端任务
- 玩家搜索页
- 玩家详情页
- 邀请码管理页
- 封禁操作页
- 服务器状态总览页

### 后端任务
- Admin API
- `ban_records` / `ban_targets`
- 审计日志
- 解绑审核流程
- 论坛封禁同步
- 游戏服封禁同步

### 验收标准
- 管理员可按 UID / 游戏 ID 搜玩家
- 管理员可完成封禁/解封
- 封禁能同步到主站、论坛、游戏服
- 关键操作有审计记录

---

## 7. Milestone 6：多游戏扩展与上线准备（第 11 周+）

### 目标
在首个 PoC 跑通后，扩展第二个、第三个游戏并补齐上线要求。

### 包含内容
- 多游戏插件接入
- 规则配置完善
- 风控补强
- 稳定性测试
- 部署与运维完善
- 监控告警
- 正式发布准备

---

## 8. 团队任务拆分建议

### 前端团队
负责：
- 官网与主站前台
- 用户中心
- 绑定流程页
- 商城与钱包页面
- 后台管理前端
- 主站与论坛 UI 融合

### 后端团队
负责：
- 用户与邀请体系
- 绑定体系
- 金币系统
- 兑换系统
- SSO
- 后台 API
- 论坛同步
- 封禁联动

### 插件/游戏接入团队
负责：
- 游戏插件开发
- 插件通信协议实现
- 事件上报
- 状态上报
- 发奖与封禁命令执行

### DevOps / 运维
负责：
- 环境搭建
- 数据库与 Redis 部署
- Discourse 部署
- 监控与日志
- CI/CD

---

## 9. 优先级建议

### P0
- 邀请制注册模型
- 游戏内绑定闭环
- 金币账本
- 首个游戏 PoC
- Discourse SSO

### P1
- 商城兑换
- 后台玩家管理
- 封禁联动
- 服务器状态展示

### P2
- 多游戏扩展
- 高级奖励规则
- 高级风控
- 更复杂的运营活动系统
