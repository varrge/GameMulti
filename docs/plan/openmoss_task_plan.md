# OpenMOSS 任务拆分与执行规划

## 1. 目标

本文档用于把当前项目规划进一步落成可在 OpenMOSS 中执行的任务体系。

目标包括：

1. 明确父任务与子任务如何拆分
2. 明确哪些任务可以并行、哪些任务存在依赖
3. 明确每条任务线最终交付物
4. 明确所有执行完成后，成果应统一提交回仓库

---

## 2. 任务拆分原则

本项目不按“前端一整块、后端一整块”粗暴拆分，而按业务闭环拆分。

采用如下 5 条主线：

1. 架构与基线定稿
2. 邀请制账户与绑定闭环
3. 首个 PoC 游戏插件接入闭环
4. 金币与商城闭环
5. 论坛与后台运营闭环

每条主线拆成多个可独立完成、可单独验收的子任务。

## 9. 执行完成后的统一交付规则

所有执行子任务在完成后，必须将以下内容统一提交回仓库：

1. 代码改动
2. 文档改动
3. 相关测试或验证说明
4. 若为设计/规划类任务，则提交到 `docs/plan/` 或对应约定目录

统一要求：

- 提交前自行检查改动完整性
- 提交信息使用规范 commit message
- 若任务涉及多个仓库，则在任务描述中明确提交目标仓库
- 若任务是规划/设计类任务，必须同步更新仓库内文档，而不是只停留在聊天输出
